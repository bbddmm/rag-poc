# フロントエンド レビュー観点

## React / Next.js

- **hooksのルール** — `useEffect` の依存配列が正しいか、不足・過剰がないか
- **再レンダリング** — 不要な再レンダリングを引き起こすパターンがないか（オブジェクトリテラルのインライン生成、関数のインライン定義等）
- **React Compiler** — React Compilerが有効なため、手動の `useMemo`/`useCallback` は原則不要。使用している場合は理由を確認する
- **"use client"** — CSR前提の構成。Server Componentを使う場合は意図を確認する
- **App Router** — `app/` ディレクトリはルーティングのみに使用し、ビジネスロジックは `features/` に配置しているか

## shadcn/ui

- **コンポーネントの使用** — shadcn/uiに該当コンポーネントがある場合、自前実装していないか
- **スタイル** — Tailwind CSSのユーティリティクラスを使用しているか、インラインスタイルやCSS Modulesを使っていないか

## 状態管理

- **Zustand** — クライアント状態（UI状態）に使用しているか。サーバー状態と混在していないか
- **TanStack Query** — サーバー状態の管理に使用しているか。キャッシュキーが適切か

## API通信

- **Hono RPC** — 型安全なAPIクライアントを使用しているか。`fetch` を直接使っていないか
- **エラーハンドリング** — APIエラー時のUI表示が適切か（ローディング、エラー、空状態）

## Storybook

- **Story** — UIコンポーネントに対応するStoryが作成・更新されているか
- **play関数** — インタラクションを伴うコンポーネントの場合、play関数でインタラクションテストが書かれているか

## アクセシビリティ

- **セマンティックHTML** — 適切なHTML要素を使用しているか（`div` の乱用がないか）
- **キーボード操作** — インタラクティブ要素がキーボードで操作可能か
