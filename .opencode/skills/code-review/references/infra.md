# インフラ レビュー観点

## SST v3 (Ion)

- **リソース定義** — リソース名が環境（dev/prod等）ごとに一意になるか
- **環境分離** — ステージ（dev/staging/prod）ごとの設定が適切に分離されているか
- **シークレット** — ハードコードされたシークレットがないか。SST Secretsまたは環境変数を使用しているか

## IAM

- **最小権限の原則** — Lambda等のIAMロールに過剰な権限（`*`）が付与されていないか
- **リソーススコープ** — 権限が特定のリソースARNにスコープされているか

## Lambda

- **タイムアウト** — 適切なタイムアウト値が設定されているか（デフォルト3秒では短すぎる場合がある）
- **メモリ** — メモリサイズが適切か（コストとパフォーマンスのバランス）
- **環境変数** — 必要な環境変数がすべて設定されているか

## DynamoDB

- **キャパシティ** — オンデマンドモードが設定されているか（PoC段階ではオンデマンド推奨）
- **TTL** — 不要なデータの自動削除が必要な場合、TTLが設定されているか

## コスト

- **Free Tier** — 可能な限りFree Tier内に収まる設定になっているか（PoC段階）
- **不要なリソース** — 使われていないリソース定義が残っていないか
