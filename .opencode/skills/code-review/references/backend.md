# バックエンド レビュー観点

## Hono

- **ルーティング** — RESTful なパス設計になっているか
- **バリデーション** — リクエストボディ・パラメータのバリデーションが行われているか
- **ミドルウェア** — 認証ミドルウェアが必要なエンドポイントに適用されているか
- **レスポンス** — 適切なHTTPステータスコードを返しているか（200/201/400/401/403/404/500等）

## DynamoDB / ElectroDB

- **アクセスパターン** — クエリがPK/SKを使用した効率的なアクセスパターンになっているか。Scanを使用していないか
- **Collection** — 同じPKを持つエンティティ間のCollection定義が適切か
- **GSI** — GSIの追加が本当に必要か、既存のインデックスで対応できないか
- **トランザクション** — 複数エンティティの更新が必要な場合、TransactWriteを使用しているか

## DI (tsyringe)

- **依存注入** — `@injectable()` デコレータが付与されているか
- **スコープ** — シングルトンとリクエストスコープの使い分けが適切か

## 認証 (Cognito)

- **トークン検証** — JWTトークンの検証が正しく行われているか
- **認可** — エンドポイントごとのアクセス制御が適切か（自分のリソースのみ操作可能か等）

## SSE ストリーミング

- **接続管理** — クライアント切断時のクリーンアップが行われているか
- **エラー伝播** — ストリーム中のエラーがクライアントに適切に伝わるか
